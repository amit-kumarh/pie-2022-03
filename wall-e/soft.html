Software

Overview

The software program for WALL-E basically have three parts, ESP32-CAM, Blynk control-ESP32 receive, Drive&Neck and FAcial expression. All development work could be doen with arduino IDE. With support of ESP32 third party board driver and libraries.

Blynk

Blynk is an IoT application for multi platforms. Each little program has its own authentication key and you could design your own user interface inside its application. After setting it up, you will need to match the data stream, for example, joystick used merged datastream, chosse V3 for data type. For testing purpose, we also desined a serial read program for digit reading, please refer to the link below. At first we also tried Wemos D1R1 wifi board for Blynk control receiver. But due to the low power efficiency and unstability, we switched to stable esp32. Please notice, some pins on ESP32 are already connected top its on-board flash memory, you cannot use them for digital input or output.

After transmitting data to ESP32, it will process and pass data to arduino(Movement control) in form of a four-digit binary number For eample(forward 1000, Backward 0100, Left 0010, right 0001)

ESP32-CAM

The ESP32 CAM uses high resolution OV2640 camera for video shooting. After flashing the code into ESP32-CAM, it will automatically connect to the network with written in information. PLease open serial monitor after flashing, IP address of CAM will be listed. Log into the ip address within the same network(so that you don't need NAT), There's neat and clean page for video send back. Face detection and recognition are based on pretrained models.
